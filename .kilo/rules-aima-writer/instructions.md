# AIMA Writer 规则

## 首次使用说明

**重要**: 首次切换到此模式时，请手动选择模型 `kimi-for-coding/k2.5`
Kilo 会记住此选择（Sticky Models 机制）

## 工作类型

### 1. 开发日志
位置: `.ai/development-log/`

格式见 AGENTS.md 第 9 节

### 2. API 文档
```markdown
## {原子单元名称}

### 描述
[功能描述]

### 输入参数
| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| xxx | string | 是 | xxx |

### 输出
```json
{
  "field": "value"
}
```

### 示例
```bash
# 示例命令
```

### 错误码
| 错误码 | 说明 |
|--------|------|
| XXX | xxx |
```

### 3. 测试设计
```markdown
## 测试用例: [名称]

### 场景
[描述]

### 输入
```json
{ "input": "data" }
```

### 预期输出
```json
{ "expected": "output" }
```

### 边界条件
- 空输入
- 最大值
- 无效输入
```

## 写作原则

1. **清晰** - 避免歧义
2. **简洁** - 不冗余
3. **完整** - 覆盖必要信息
4. **结构化** - 易于浏览

## 日志文件命名

```
{日期}-{任务描述}.md
示例: 2026-02-16-implement-model-pull.md
```
